# PharmaFix: Micro-Stage Roadmap

This roadmap breaks down the construction of PharmaFix into atomic, testable stages.

## Stage 1: Core Initialization & Design System (Foundational)
- [ ] **1.1. Project Skeleton**: Flutter init with professional structure.
- [ ] **1.2. Analysis Rules**: Configure `analysis_options.yaml` for strict linting.
- [ ] **1.3. Theme Engine**: Implement Dark/Light mode tokens (Colors, Typography).
- [ ] **1.4. Shared Components (Atoms)**: Buttons, Text Fields, Custom App Bars.

## Stage 2: Authentication & User Profile (Security)
- [ ] **2.1. Auth State Management**: Setup BLoC/Cubit for user sessions.
- [ ] **2.2. Login/Sign-up UI**: Stunning forms with validation.
- [ ] **2.3. Profile Screen**: Customizable user settings.

## Stage 3: Medicine Discovery (Core Feature)
- [ ] **3.1. API/Data Layer**: Setup Dio and repository pattern.
- [ ] **3.2. Search Logic**: Debounced search with filtering.
- [ ] **3.3. Results Feed**: List and Grid views for medicine items.
- [ ] **3.4. Medicine Detail View**: Deep dive into medicine details (dosage, effects).

## Stage 4: Stock & Pharmacy Services (Advanced)
- [ ] **4.1. Pharmacy Dashboard**: Inventory management UI.
- [ ] **4.2. Map Integration**: Using Google Maps for location services.
- [ ] **4.3. Prescription Upload**: Camera/Gallery integration.

## Stage 5: Polishing & Deployment (Final)
- [ ] **5.1. Motion Design**: Fine-tune transitions and micro-animations.
- [ ] **5.2. Localization**: Fully Arabic/English support (RTL/LTR).
- [ ] **5.3. Build & QA**: Testing and production builds.
